# 企业工单流转平台

一个模拟企业内部工单流转的后端系统，核心解决多人抢单并发竞争、工单状态机流转、超时释放、SLA 升级通知等问题。主体为 Java/Spring Boot 后端项目，嵌入 LLM 智能 triage 作为亮点。

## 语言

### 核心概念

**工单**:
由提交人创建、需要被处理的一条工作请求。工单有其明确的生命周期和状态流转规则。
_Avoid_: 任务、需求单

**工单编号**:
工单的业务标识号，格式 `WO-20260607-00001`（前缀 + 日期 + 当日自增序号），非自增主键，对外展示和查询的主键。
_Avoid_: ID、工单号

**抢单**:
多个处理人同时对同一张待分配工单发起"接单"操作，只有一人能成功。这是并发竞争场景。
_Avoid_: 认领、抢占

**验收**:
处理人完成处理后，提交人确认处理结果是否通过的环节。不通过则驳回回退。
_Avoid_: 审核、审批

**SLA**:
工单的服务等级协议，定义不同优先级的工单必须在多长时间内被接单或处理完毕。超时则自动升级通知管理员。
_Avoid_: 时效要求、时限

**工单 Triage**:
工单提交时，由 LLM 自动分析工单内容，判断工单类型和优先级，减少人工分派工作量。
_Avoid_: 工单分类、智能分配

**驳回**:
提交人对处理结果不满意，将工单退回处理中状态。工单有驳回次数上限（默认 3 次），达到上限后自动升级管理员介入。
_Avoid_: 退回、拒绝

### 角色与权限

**提交人**:
创建工单的人，也是最终验收工单是否处理完成的人。只能查看自己提交的工单。
_Avoid_: 申请人、发起人、普通用户

**处理人**:
有能力处理工单的角色，可以看到待分配工单池并抢单，处理分配给自己的工单。
_Avoid_: 员工、受理人

**部门主管**:
管理本部门工单的角色，可以查看部门所有工单、手动分配工单、处理升级工单。不参与日常抢单。
_Avoid_: 组长、经理

**系统管理员**:
全局管理角色，配置 SLA 规则、管理用户和角色、查看全局统计。
_Avoid_: 超级管理员、admin

**RBAC**:
基于角色的访问控制模型。用户 → 角色 → 权限三层结构。接口通过权限码拦截，而非硬编码角色判断。
_Avoid_: ACL、硬编码角色判断

**权限码**:
接口访问的最小粒度标识，如 `order:accept`（抢单）、`order:admin:stats`（全局统计）。角色是权限码的集合。
_Avoid_: 接口权限、功能权限

### 通知

**通知渠道**:
通知的传输媒介——站内信（预置）、邮件、短信、企业微信（扩展点）。通知模块通过策略模式支持多渠道路由，当前仅实现站内信。
_Avoid_: 消息方式、推送方式

**站内信**:
存储在数据库中的通知消息，用户在下次请求时拉取并展示红点提醒。不做实时推送。
_Avoid_: 系统消息、通知消息
